# Plan a complex change

Use this skill manually when a change crosses package or architecture boundaries.

1. Define the objective, non-goals, ownership boundary, and compatibility constraints.
2. Map the current implementation, tests, documentation, and upstream divergence before proposing edits.
3. Order dependent changes and name the smallest validation gate for each step.
4. Call out migrations, security risks, rollback points, and unresolved decisions.
5. Keep optional work explicitly separate from the required implementation.
